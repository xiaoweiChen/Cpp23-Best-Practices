\item

原始的 Makefile 或 Visual Studio 项目文件使得实现上述各项内容变得过于困难。请使用构建工具来帮助您维护跨平台和跨编译器的可移植性。

请像对待其他任何代码一样对待您的构建脚本。它们有自己的一套最佳实践，编写出难以维护的构建脚本，就像写出难以维护的 C++ 代码一样容易。

构建生成工具(build generators)还能通过诸如 `cmake --build` 这样的命令，帮助您抽象并简化持续构建环境，该命令能够在任何平台上执行正确的构建操作。

\begin{myTip}{练习：审视您的构建系统}
\begin{itemize}
\item
您的项目目前是否使用了构建生成工具？

\item
您的构建脚本有多久的历史了？
\end{itemize}

检查是否有当前的最佳实践需要应用。是否存在类似 `tidy` 的工具或格式化工具可用于您的构建脚本？  
回顾本书前面提到的最佳实践，思考它们如何适用于您的构建脚本。

\begin{itemize}
\item
您是否在重复编写相同的代码？

\item
是否有更高层次的抽象可用？
\end{itemize}
\end{myTip}

\begin{myTip}{}
最新版本的 CMake 增加了诸如 `--profiling-output` 之类的工具，可以帮助您查看生成器在哪些环节耗时最多。
\end{myTip}
