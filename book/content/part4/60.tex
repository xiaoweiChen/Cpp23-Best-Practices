“初始化列表”(Initializer List)在C++中是一个被过度使用的术语。“初始化列表”用于直接初始化值，而 initializer\_list 则用于将一组值传递给函数或构造函数。

\begin{myTip}{练习：理解 initializer\_list 可能带来的开销}
使用 Andreas Fertig 出色的 \href{http://cppinsights.io}{cppinsights.io} 工具来理解以下两个示例的行为：

\filename{图 121. 使用 'shared\_ptr' 的 ‘initializer\_list‘ 构造函数。}
\begin{cpp}
#include <vector>
#include <memory>

std::vector<std::shared_ptr<int>> vec{
  std::make_shared<int>(40), std::make_shared<int>(2)
};
\end{cpp}

\filename{图 122. 使用 'shared\_ptr' 构造 ‘std::array‘}
\begin{cpp}
#include <array>
#include <memory>

std::array<std::shared_ptr<int>, 2> data{
  std::make_shared<int>(40), std::make_shared<int>(2)
};
\end{cpp}

并解释它们之间的区别。如果你能做到这一点，那么你对这个问题的理解就超过了大多数 C++ 开发者。
\end{myTip}

\begin{myTip}{练习：理解为什么这段代码无法编译}
\filename{图 123. 使用 'unique\_ptr' 的 ‘initializer\_list‘ 构造}

#include <vector>
#include <memory>

std::vector<std::unique_ptr<int>> data{
  std::make_unique<int>(40), std::make_unique<int>(2)
};
\end{cpp}
\end{myTip}
